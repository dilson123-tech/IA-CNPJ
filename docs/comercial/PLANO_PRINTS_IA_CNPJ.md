# PLANO_PRINTS_IA_CNPJ

Data base: 2026-03-19  
Projeto: IA-CNPJ  
Objetivo: definir a captura oficial das telas que entrarão no PDF/apresentação comercial, sem reabrir o core técnico e sem alterar fluxos já validados.

---

## 1. Regra-mãe deste documento

Este plano existe para organizar a captura visual do IA-CNPJ com padrão comercial.

O foco aqui não é mexer no produto.
O foco aqui é decidir:

- quais telas precisam ser capturadas
- em que ordem elas devem ser capturadas
- o que precisa aparecer em cada print
- qual é o critério mínimo para considerar um print aprovado
- como preparar o material para virar PDF/apresentação comercial

---

## 2. Regras operacionais

- Não reabrir backend, auth, IA consultiva, PDF técnico, dashboard base ou fluxos já validados sem bug real.
- Não transformar este plano em sprint de redesign.
- Se alguma tela estiver funcional porém visualmente aceitável, capturar e seguir.
- Ajustes cosméticos só entram se forem pequenos, objetivos e claramente úteis para venda.
- A prioridade é fechamento comercial, não perfeccionismo técnico.

---

## 3. Objetivo do pacote de prints

O conjunto de prints deve provar visualmente que o IA-CNPJ é um produto real, profissional e pronto para uso.

O material visual deve transmitir:

- clareza
- confiabilidade
- organização
- inteligência aplicada
- utilidade para operação empresarial
- leitura executiva
- potencial comercial imediato

---

## 4. Ordem oficial sugerida de prints

1. Tela de login
2. Dashboard principal
3. Tela/listagem de empresas
4. Tela de criação de empresa
5. Tela de relatórios
6. Tela de resultado consultivo com IA
7. Tela/ação ligada ao PDF ou saída executiva
8. Tela complementar de credibilidade (caso necessário)

---

## 5. Prints obrigatórios

### Print 01 — Login
**Objetivo comercial:** mostrar acesso profissional e superfície séria de entrada.  
**O que precisa aparecer:**
- tela limpa
- branding/nome do produto
- campos organizados
- aparência profissional
**Critério de print aprovado:**
- sem erro visual
- sem dados bagunçados
- sem elementos quebrados
- layout confiável

---

### Print 02 — Dashboard principal
**Objetivo comercial:** mostrar visão geral do sistema.  
**O que precisa aparecer:**
- estrutura principal do painel
- navegação visível
- cards/resumos relevantes
- sensação de produto operacional
**Critério de print aprovado:**
- leitura rápida
- sem estado vazio constrangedor
- sem banners estranhos
- sem desalinhamentos visíveis

---

### Print 03 — Empresas
**Objetivo comercial:** mostrar gestão de empresas/clientes/CNPJs no sistema.  
**O que precisa aparecer:**
- listagem ou estrutura de empresas
- organização visual
- dados suficientes para dar credibilidade
**Critério de print aprovado:**
- tabela/lista legível
- sem poluição
- sem aparência de protótipo cru

---

### Print 04 — Criação de empresa
**Objetivo comercial:** mostrar ação real de cadastro/entrada operacional.  
**O que precisa aparecer:**
- formulário ou fluxo claro
- campos coerentes
- sensação de produto pronto para uso
**Critério de print aprovado:**
- formulário organizado
- sem excesso visual
- sem labels quebradas

---

### Print 05 — Relatórios
**Objetivo comercial:** mostrar capacidade analítica do sistema.  
**O que precisa aparecer:**
- área de relatórios
- organização da saída
- leitura executiva mínima
**Critério de print aprovado:**
- tela compreensível
- sem ruído
- conteúdo com cara de valor real

---

### Print 06 — Consultoria com IA
**Objetivo comercial:** provar o diferencial do produto.  
**O que precisa aparecer:**
- resultado consultivo
- insights ou leitura analítica
- inteligência aplicada ao contexto empresarial
**Critério de print aprovado:**
- resposta clara
- valor percebido imediato
- sem texto quebrado
- sem cara de experimento

---

### Print 07 — PDF / saída executiva
**Objetivo comercial:** mostrar entregável premium.  
**O que precisa aparecer:**
- ação de geração ou evidência do PDF
- saída com aparência formal/executiva
- material utilizável comercialmente
**Critério de print aprovado:**
- aparência séria
- legibilidade
- sensação de entrega final real

---

### Print 08 — Print complementar de credibilidade
**Objetivo comercial:** reforçar robustez do produto.  
**Pode ser:**
- detalhe extra do dashboard
- tela de transações
- tela de fluxo operacional
- tela que reforce valor para cliente
**Critério de print aprovado:**
- deve agregar valor narrativo
- não pode repetir visualmente os prints anteriores sem necessidade

---

## 6. Checklist de preparação antes de capturar

Antes de qualquer print, validar:

- backend no ar
- frontend no ar
- login funcionando
- sessão ativa
- navegação abrindo corretamente
- dados minimamente apresentáveis
- ausência de mensagens de erro
- ausência de elementos visuais quebrados
- resolução da tela adequada
- navegador limpo e sem distrações

---

## 7. Critérios de reprovação de print

Um print deve ser refeito se tiver:

- loading estranho
- banner indevido
- dado desalinhado
- texto quebrado
- área vazia constrangedora
- aparência de sistema inacabado
- excesso de informação irrelevante
- erros visuais perceptíveis
- inconsistência de idioma
- sensação de ambiente improvisado

---

## 8. Critério de “print aprovado”

Um print está aprovado quando:

- comunica rápido
- está limpo
- reforça credibilidade
- ajuda a vender o produto
- parece parte de um sistema real
- pode entrar no PDF sem vergonha comercial

---

## 9. Uso prático deste plano

Este documento será usado para:

1. organizar a ordem da sessão de captura
2. evitar prints aleatórios
3. garantir coerência no PDF comercial
4. montar apresentação com narrativa visual
5. acelerar o fechamento do pacote comercial final

---

## 10. Próximo passo após este documento

Depois de fechar este plano, a próxima etapa é:

- validar o ambiente visualmente
- capturar as telas na ordem definida
- selecionar as melhores versões
- encaixar os prints na estrutura do PDF/apresentação comercial
